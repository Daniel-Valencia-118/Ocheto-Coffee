II. ANÁLISIS Y DISEÑO ESTRUCTURADO

Diagrama de Contexto Representación de alto nivel que muestra el sistema como un único proceso y sus interacciones con entidades externas.

El sistema de gestión de cafetería interactúa con:

  • Clientes (realizan pedidos)
  • Personal de cocina (preparan pedidos)
  • Administrador (genera reportes)
  • Proveedores (suministran ingredientes)

Diagrama de Flujo de Datos (DFD) Nivel 0 Descomposición del diagrama de contexto que muestra los procesos principales del sistema.

Procesos principales:

  • Gestión de Pedidos
  • Gestión de Inventario
  • Gestión de Clientes
  • Generación de Reportes

Diagrama de Flujo de Datos (DFD) Nivel 1 Desglose detallado de cada proceso principal mostrando subprocesos y flujos de datos.

Para el proceso de Gestión de Pedidos:

  • Registrar pedido
  • Verificar disponibilidad
  • Calcular total
  • Actualizar inventario

Diccionario de Datos Definición detallada de todos los elementos de datos utilizados en el sistema.

Ejemplos:

  • Pedido = NúmeroPedido + Fecha + Cliente + ListaProductos + Total
  • Producto = CódigoProducto + Nombre + Precio + Categoría
  • Cliente = IDCliente + Nombre + Teléfono + Correo
