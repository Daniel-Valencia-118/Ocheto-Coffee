Análisis Orientado a Objetos

Modelado UML del sistema de gestión para la cafetería OCHETO utilizando el paradigma orientado a objetos. Incluye diagramas de clases, casos de uso y especificaciones detalladas.

Análisis y Diseño Orientado a Objetos

Diagrama de Clases

Representación estática de las clases del sistema, sus atributos, operaciones y relaciones.

El diagrama de clases UML representa la estructura estática del sistema, mostrando las clases principales, sus atributos, métodos y las relaciones entre ellas (herencia, asociación, agregación, composición).

Diagrama de Clases UML

Estructura estática del sistema con relaciones entre clases

Clase: Producto

Representa los productos de la cafetería con atributos como nombre, precio, categoría y stock.

Clase: Pedido

Gestiona los pedidos de los clientes con relación a productos, cantidades y estado.

Clase: Usuario

Base para empleados y clientes con atributos de autenticación y permisos.

Este diagrama muestra los casos de uso desde la perspectiva del negocio, identificando los principales procesos que el sistema debe soportar y los actores involucrados en cada uno.

Procesos comerciales principales y actores involucrados

Especificación de Casos de Uso (Texto)

Descripción detallada en formato textual de cada caso de uso del sistema.

Los casos de uso se especifican detalladamente en formato textual para cada funcionalidad del sistema, incluyendo precondiciones, flujo principal, flujos alternativos y postcondiciones.

Caso de Uso: Realizar Pedido

Actor Principal:
Cliente / Cajero
Precondiciones:
  • El cliente debe estar autenticado en el sistema
  • Debe haber productos disponibles en el inventario
  • El cajero debe tener permisos para realizar ventas
Flujo Principal:
  1. El cliente selecciona los productos deseados
  2. El sistema valida la disponibilidad de cada producto
  3. El sistema calcula el total de la compra
  4. El cliente realiza el pago
  5. El sistema registra la venta y actualiza el inventario
  6. El sistema genera el comprobante de venta
Postcondiciones:
  • El pedido queda registrado en el sistema
  • El inventario se actualiza con las cantidades vendidas
  • Se genera un registro en el historial de ventas
  • El cliente recibe su comprobante de pago
